Building SystemJS Plugins

Yay!

By Matthew Phillips of Bitovi

Examples

With SystemJS you can...

  1. Load just about anything that can be converted into JavaScript.
  2. Invent new syntax rules for loading.
  3. Preload images so they are freshly cached.
  4. Create special types of behaviors in your module loader.

Plugins vs. Extensions

Plugins

  • Are applied to modules during the module-loading process.
  • Can implement the locate, fetch, translate, and instantiate hooks.
  • Require that the user explicitly use them on each module.



steal('package.json!json', function(pkg) {
  console.log('Version is:', pkg.version);
});

Extensions

  • Extend the functionality of a Loader.
  • Gets applied to every module that runs through the loading process.
  • Have access to all hooks plugins have plus normalize.
  • Currently are simply just wrapper functions.

function enforceHttps(loader) {
  var loaderLocate = loader.locate;
  loader.locate = function(load) {
    return loaderLocate.call(this, load).then(function(address) {
      if(address.indexOf('https') !== 0) {
        throw 'Loading from non-https protocol is not allowed!';
      }
      return address;
    });
  };
}

enforceHttps(System);

SystemJS Hooks

Normalize

  • Converts a module name to the fully normalized module name.
  • can/map becomes can/map/map
  • ./utils becomes app/utils when in the app folder.

Locate

  • Given a load (essentially the name) find the address of the module.
  • Overrides the location of the module.
  • Load from a different protocol (enforce https?).
  • Attach query parameters to the request.

Fetch

  • Gives you full control over fetching the module.
  • If you want to load the module in some other method than XHR.

Translate

  • Convert the source of a load to some other representation.
  • For example, transpiling to JavaScript.
  • Returning undefined will continue through the loading process as if load.source has been modified.
  • Returning a source will exit the loading process.

Instantiate

  • Last chance to override the default instantiate behavior.
  • Can override execute at this point.
  • Set the module's format.
  • Declare the module's dependencies.

All SystemJS hooks can return a promise. This is powerful because it means async behavior can happen in the hook, and hooks can call other hooks as needed (normalize can use System.import).

Building Up

The power of these hooks is that they allow you to build your own custom loader by stacking extensions on top of each other.

SystemJS

Is (pretty much) a set of extensions built on top of the ES6 Module Loader.

It adds..

  • Support for AMD, CommonJS, and Global modules
  • Map configuration
  • Meta configuration
  • Support for plugins
  • Bundles
  • System.register
  • Fetching through script tags
  • Versioning (load a specific version of jQuery)

Steal

Is a set of extensions built on top of SystemJS.

A backwards compability layer with Legacy Steal.

It adds..

  • Steal's module naming convention. foo normalizes to foo/foo.
  • Extension configuration (instead of foo.stache!stache just foo.stache!).
  • Forward-slash extension (foo/ normalizes to foo/foo).
  • The Steal format: steal('can', function(){}).
